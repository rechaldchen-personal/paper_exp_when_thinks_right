## 1. Run 1 is withdrawn

Run 1 (2026-07-17) is not usable and its numbers must not be quoted. Two
independent classes of defect:

**Stimuli.** Only 31 of 78 pairs held target and distractor constant across
conditions. All 28 arithmetic pairs *swapped* them, so the paired test
compared different questions with different answers, confounding the framing
manipulation with token identity and frequency. 15 of 23 garden-path prompts
ended with the answer word, leaving nothing to predict at the query position.
Arithmetic answers were also unreachable in practice: Qwen3 tokenizes ' 10' as
[220, 16, 15], so the model's top-1 was the bare-space token on 55/56 items —
it was correctly beginning a digit answer while the stimuli keyed on English
number words.

**Analysis.** The dev/holdout split was not reproducible (`list(set(...))` plus
per-process string hashing defeated `seed=42`); the test was two-tailed
although §1 of the original document specifies one-tailed; and the p-value
depended on the scipy version. Full detail in `out/ANALYSIS_NOTE.md`.

## 2. Amendment A1 — stimulus set replaced

The stimulus set is regenerated by `build_stimuli.py` and enforced by
`validate_stimuli.py` (rules R1–R6). Binding requirements:

- **R1 matched pairs.** Within a `pair_id`, target and distractor are identical
  across conditions; only the prompt framing differs. This is what makes the
  design paired, and it is now a hard validation failure if violated.
- **R2 no answer leakage.** The prompt may not end with the target or
  distractor word.
- **R3 single token.** Target and distractor must each be one token in the
  leading-space form.
- **R6 no answer in context.** A few-shot query may not repeat a worked example.

Composition: **72 matched pairs (24 factual, 24 arithmetic, 24 garden-path)
plus 12 hard controls**, replacing 78 pairs of which 31 were valid and 3 hard
controls. Families are deliberately balanced so the pooled test is not
dominated by arithmetic, the family whose behavioural accuracy is least
certain.

This supersedes §7 of the original document (sample size). Expected complete
pairs per test rise from n≈6–11 to n≈40–70, which is the point: the original
run could not have reached p<0.05 for H1/H3 at n=5–6 regardless of effect size.

## 3. Amendment A2 — statistical test implementation corrected

No change of test. The original document specified a paired Wilcoxon
signed-rank test, one-tailed, on (false_lead − straightforward). That is what is
now actually computed:

- **One-tailed** (`alternative="greater"`), as originally registered. The run-1
  code used scipy's two-sided default.
- **Exact**, computed in-repo by subset-sum DP over mid-ranks, dropping zero
  differences (standard `zero_method="wilcox"`). This removes the scipy-version
  dependence and is correct under ties, where scipy's own `method="exact"` is
  not. Verified against brute-force enumeration and against scipy on tie-free
  data up to n=30 (`verify_env.py` re-checks this on every setup).
- **Reproducible split**: pair IDs are sorted before shuffling, so `seed=42`
  fixes the split as the original document intended.

Both one- and two-tailed p-values are recorded in `report.json` regardless.

## 4. Amendment A3 — confirmatory 2AFC metrics added

**The primary metrics are unchanged.** H1, H2, H3 remain exactly as registered,
over the global argmax, and remain the headline tests.

Added alongside them, a **confirmatory readout restricted to {target,
distractor}**, motivated by an ambiguity the registered metrics cannot resolve:
`oscillation` counts changes in the global top-1 identity, which rises both
when the model wavers *between the candidate answers* and when it churns among
unrelated vocabulary. Only the former supports the "internal revision" reading
that the paper's framing rests on.

Definitions mirror the primary ones exactly (same band, same stability rule,
same post-ℓ_H window), differing only in the readout:

| Confirmatory | Definition |
|---|---|
| `l_star_2afc` | first band layer where logp(target) > logp(distractor) and stays so through the end of the band |
| `gap_2afc` | `l_star_2afc − ℓ_H` |
| `oscillation_2afc` | count of flips in the sign of logp(target) − logp(distractor) across band layers after ℓ_H |

Directional predictions are the same as H1–H3. Support criterion is the same:
p < 0.05 one-tailed with median difference in the predicted direction.

`l_star_2afc` additionally has materially better coverage than `l_star`
(defined for 21/32 vs 11/32 holdout pairs in the run-1 pilot), because it does
not require the target to beat the entire vocabulary.

## 5. Interpretation rules, fixed in advance

For H2 specifically, the joint outcome determines what may be claimed. These
readings are committed to now, before the data:

| Primary `oscillation` | Confirmatory `oscillation_2afc` | Permitted claim |
|---|---|---|
| significant | significant | Internal revision **between the candidate answers**. The paper's current framing is supported. |
| significant | null | Increased top-1 churn under false lead that is **not** attributable to target/distractor competition. Must be reported as unattributed churn; the "revision between candidates" language must be removed from Abstract and Discussion. |
| null | significant | Competition between candidates without global top-1 instability. Reframe as a sub-threshold preference effect. |
| null | null | H2 not supported. |

The same logic applies to H1/H3 versus their confirmatory counterparts:
agreement strengthens the claim, divergence caps it at the weaker reading.

## 6. Reporting commitment

Both the primary and the confirmatory tests will be reported in full — all
six p-values, effect sizes, and n — **whichever way they come out**, in the
Results section and not relegated to an appendix. The run-1 pilot values in §0
will be reported next to them. If the confirmatory tests contradict the
primary ones, that contradiction is the finding and will be stated as such.

## 7. Unchanged from the original pre-registration

Workspace band [0.25, 0.90]; θ = 80th percentile of straightforward ΔH computed
on the dev set only; 60/40 dev/holdout split by pair with seed 42; hard controls
as the specificity check; band and θ sensitivity analyses; logit-lens secondary
readout; the H4 CoT extension as optional and descriptive.

## 8. Stop conditions

Before any hypothesis test on run 2, the behavioural pre-screen of §5.4 is run
**per family**. If a family's straightforward condition does not put the target
top-1 for a clear majority of items, that family's stimuli are fixed and the
traces recollected before analysis — the failure is not analyzed around. Run 1
should have stopped here: arithmetic was 1/56 and was analyzed anyway.
